## Ответ для отклика

**1. Краткое описание проекта**

Учебный full-stack проект SupportBot: пайплайн `Telegram Bot -> Supabase Edge Function -> Supabase Postgres -> Next.js admin panel`.

Что реализовано:
- приём webhook-сообщений из Telegram;
- валидация входящих данных в Edge Function на Deno;
- сохранение сообщений в таблицу `messages` в Supabase;
- отправка автоматического ответа пользователю;
- админ-панель на Next.js для просмотра сообщений, фильтрации по ботам и чатам, удаления отдельных сообщений и целых диалогов.

Мой вклад:
- спроектировал и собрал полный поток данных от Telegram до админки;
- реализовал webhook-обработчик, разбор update, валидацию, сохранение сообщений и отправку ответа;
- настроил структуру БД и миграции;
- сделал интерфейс админки и серверные route handlers для административных действий;
- держал решение простым, без лишних абстракций, чтобы система была прозрачной и легко поддерживаемой.
